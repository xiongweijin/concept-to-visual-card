# Concept Extraction

Goal: turn an abstract concept into a visualizable structure.

## Extraction Steps

1. Name the concept in the user's language.
2. Strip domain jargon.
3. Identify the mechanism: what changes, causes, constrains, maps, or reverses what?
4. Choose the structural type.
5. Write one plain sentence that a non-specialist can understand.

## Structural Types

| Type | Pattern | Visual Direction |
|---|---|---|
| 分解 | Whole breaks into meaningful parts | exploded view, layers, parts map |
| 映射 | One structure corresponds to another | map, mirror, parallel columns |
| 反馈 | Output loops back as input | loop arrows, echo, compounding cycle |
| 涌现 | Simple interactions create complex result | many small pieces forming a larger pattern |
| 阈值 | Gradual accumulation triggers sudden change | meter, tipping point, phase shift |
| 约束 | Rules or boundaries shape behavior | frame, fence, channel, rule board |
| 反转 | Intuition is backwards | before/after, flipped perspective |
| 权衡 | Gaining one thing costs another | balance, sliders, trade-off scale |
| 选择 | Repeated filtering keeps some and removes others | sieve, funnel, selection path |
| 自指 | The system refers to itself | mirror loop, recursive frame |

## Output Shape

```markdown
- 概念名称：
- 核心机制：
- 去术语化解释：
- 结构类型：
- 视觉化机会：
```

## Quality Check

- The explanation should avoid unnecessary technical terms.
- The visual opportunity should come from structure, not surface metaphor.
- If multiple concepts are mixed together, split them before creating the image.
